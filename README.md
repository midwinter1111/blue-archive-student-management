# ブルアカ育成管理アプリ

ブルーアーカイブ（PC版）の生徒育成状況を一括管理するローカルWebアプリです。  
ゲーム画面をキャプチャしてOCRで育成データを自動抽出し、一覧・フィルタリングできます。

---

## 動作環境

| 項目 | 要件 |
|---|---|
| OS | Windows 10 / 11 |
| Python | 3.12 以上 |
| Node.js | 18 以上 |
| ゲーム | ブルーアーカイブ 公式PCクライアント |

---

## 初回セットアップ

### 1. Python のインストール（未インストールの場合）

ターミナル（コマンドプロンプト / PowerShell）で以下を実行します。

```
winget install Python.Python.3.12 --accept-package-agreements --accept-source-agreements
```

インストール後、**ターミナルを再起動**してから以下で確認してください。

```
python --version
pip --version
```

> **うまく動かない場合**  
> 設定 → アプリ → アプリの詳細設定 → アプリ実行エイリアス で  
> 「python.exe」「python3.exe」をオフにしてから再度確認してください。

---

### 2. Node.js のインストール（未インストールの場合）

```
winget install OpenJS.NodeJS.LTS --accept-package-agreements --accept-source-agreements
```

インストール後ターミナルを再起動し、確認します。

```
node --version
npm --version
```

---

### 3. バックエンドのセットアップ

```
cd backend
pip install -r requirements.txt
```

> **注意（初回のみ）**  
> `easyocr` のインストール後、サーバー起動時に日本語OCRモデルを自動ダウンロードします。  
> 約 **2GB** の容量が必要で、初回起動に数分かかります（2回目以降は不要）。

---

### 4. フロントエンドのセットアップ

```
cd frontend
npm install
```

---

## 起動方法

毎回使うときは、**バックエンドとフロントエンドの両方を起動**する必要があります。  
ターミナルを2つ開いて、それぞれで以下を実行してください。

### ターミナル① バックエンド起動

```
cd backend
python main.py
```

正常に起動すると以下のように表示されます。

```
⌨️  Ctrl+Shift+S でキャプチャ開始
✅ サーバー起動完了
📸 ホットキー Ctrl+Shift+S でキャプチャ
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### ターミナル② フロントエンド起動

```
cd frontend
npm run dev
```

正常に起動すると以下のように表示されます。

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

### ブラウザでアクセス

ブラウザで以下を開きます。

```
http://localhost:5173
```

## 使い方

### 画面構成

アプリは2つのモードで構成されています。

| タブ | 説明 |
|---|---|
| 📋 一覧 | 全生徒の育成状況を表示・フィルタリング |
| 📸 登録 | ゲーム画面をキャプチャして育成状況を登録 |

---

### 育成状況を登録する

1. ブルーアーカイブを起動し、登録したい生徒の **「基本情報」タブ** を開く

   ![基本情報タブ]

2. アプリの **「📸 登録」タブ** を開く

3. 以下のどちらかでキャプチャを実行する

   | 方法 | 操作 |
   |---|---|
   | ボタン操作 | 「📸 今すぐキャプチャ」ボタンをクリック |
   | ホットキー | `Ctrl + Shift + S` を押して「⌨️ Ctrl+Shift+S 待機」ボタンをクリック後キー入力 |

4. キャプチャ結果が表示される。内容を確認し、誤認識があれば手動で修正する

5. 「💾 保存」をクリックして登録完了

> **ポイント**  
> - 「基本情報」タブ以外のページ（レベルアップ、神秘解放など）ではキャプチャが機能しません  
> - OCRの誤認識が多い場合は手動で数値を修正して保存してください

---

### 育成状況を一覧で確認する

「📋 一覧」タブで以下のフィルターが使えます。

| フィルター | 内容 |
|---|---|
| 名前検索 | 生徒名の部分一致で絞り込み |
| 加入状態 | 「加入済みのみ」「未加入のみ」で切り替え |
| スキル | スキルがMAX未達の生徒のみ表示 |
| 装備 | 装備T10未達の生徒のみ表示 |
| 凸/固有 | 星ランク・固有武器レベルの範囲で絞り込み（1〜9） |

列ヘッダーをクリックするとソートできます。

## データの見方

### 凸・固有の表記

| 表示 | 意味 |
|---|---|
| 星1 〜 星4 | 固有武器なし（星ランク1〜4） |
| 固有1 〜 固有5 | 固有武器あり（★5解放済み、固有武器Lv1〜5） |

### スキルの表記

| 表示 | 意味 |
|---|---|
| 1 〜 9 | スキルレベル |
| M | MAX（レベル10） |

### WorkBook（WB）の表記

| 表示 | 意味 |
|---|---|
| 0 または - | 未振り |
| 1 〜 25 | 振ったレベル |

H = HP / A = 攻撃力 / R = 治癒力

## 生徒の手動追加

マスターリストにない生徒（新実装・コラボなど）を追加したい場合は、  
[backend/students_master.py](backend/students_master.py) の `STUDENTS_MASTER` リストにゲーム内表記の名前を追記してください。

```python
STUDENTS_MASTER = [
    ...
    "新しい生徒名",  # ← ここに追加
]
```

追加後はバックエンドを再起動してください。

## トラブルシューティング

### `pip` が見つからない

ターミナルを再起動してください。それでも解決しない場合は Python の PATH が通っていない可能性があります。  
→ セットアップ手順 1 の「うまく動かない場合」を参照してください。

### ゲームウィンドウが検出されない（画面右上が赤い）

- ゲームが起動しているか確認してください
- ゲームのウィンドウタイトルが「ブルーアーカイブ」になっているか確認してください
- バックエンドを再起動してください

### 生徒名が正しく認識されない

OCRの誤認識です。キャプチャ後の確認画面で名前を手動修正して保存してください。  
正しい名前を入力するとマスターリストと照合して保存されます。

### EasyOCRモデルのダウンロードが止まっている

初回起動時に約2GBのモデルをダウンロードします。ネットワーク環境によっては時間がかかります。  
完了するまでそのままお待ちください（ターミナルに何も表示されない時間が続くことがあります）。

### スキルや装備の数値がずれる

ゲームウィンドウサイズが基準解像度（1456×784）と大きく異なる場合、OCRの読み取り位置がずれることがあります。  
[backend/extractor.py](backend/extractor.py) の `COORDS` の値を調整することで対応できます（上級者向け）。
